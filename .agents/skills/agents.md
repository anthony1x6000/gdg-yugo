# Agent Instructions

When operating within this repository, all AI agents, coding assistants, and automated tools MUST always check and utilize the skills defined in this directory.

## Available Skills

- **Humanizer** (`humanizer-skill.md`): When writing, editing, or reviewing text (including documentation and markdown files), you must automatically apply the guidelines in `humanizer-skill.md` to remove signs of AI-generated patterns and ensure the text sounds natural, human-written, and matches the repository's tone.

## Core Directives for Agents

1. **Always Check for Skills**: Before executing a text generation, refactoring, or documentation task, review the skill documents in this directory.
2. **Apply Automatically**: You do not need explicit permission to apply these skills. Automatically apply the constraints, patterns, and style guides defined in the skill documents to your outputs.
3. **Use the Specified Process**: If a skill defines a specific workflow (e.g., the Humanizer's "Identify, Rewrite, Preserve, Maintain Voice, Add Soul, Audit" process), you must follow those steps implicitly before presenting your final output.
4. **Voice Matching**: Where applicable, look for writing samples in the repository to calibrate your output voice instead of defaulting to a generic AI tone.

Failure to adhere to these skills will result in rejected outputs.
